## Design Context

### Users
Single user (the developer/owner). Used as a personal local AI assistant — the context is quiet, focused work sessions. Not a product to impress others; a tool to feel good using every day.

### Brand Personality
**Quiet, organic, introspective.** ARIA should feel like Obsidian's dark theme — comfortable to live in for hours. Not a hacker terminal. Not a neon-cyber dashboard. Something that breathes.

### Aesthetic Direction
- Obsidian-inspired: very dark backgrounds with a slight violet warmth (not cold blue-black), muted lavender accents instead of electric purple
- "Flowy not cyber" — smooth transitions, soft corners, organic type, no harsh neons
- Dark theme (correct for single-user local tool, used at a desk coding)
- Anti-references: DO NOT make it look like a crypto/DeFi dashboard, a hacker terminal, or a generic AI product landing page

### Fonts
- Display/labels: **Bricolage Grotesque** — editorial, organic, has genuine character
- Body/UI: **Figtree** — warm humanist grotesque, reads well at small sizes
- Monospace (code only): system stack — `'Cascadia Code', 'Fira Code', ui-monospace, monospace`

### Color System (OKLCH)
- Backgrounds: tinted dark charcoal toward violet hue 285
- Accent: muted lavender `oklch(67% 0.13 292)` — NOT electric purple
- Secondaries: desaturated teal, amber, blue, green, red
- Text: warm off-white, never pure white

### Design Principles
1. **Quiet confidence** — density without aggression; information-rich but never loud
2. **Organic warmth** — slightly warm neutrals, soft curves, gentle transitions
3. **Earned attention** — color is used sparingly; when something is colored, it means something
4. **Flow over flash** — 200ms ease-out transitions throughout; nothing bounces or jolts
5. **Personal scale** — sized for one person who knows the tool; no onboarding theater
6. **No marketing copy in the UI** — this is a tool, not a product page; avoid taglines, feature pills, and self-promotional text

## Current UI Structure

### Layout
3-column grid: `256px | 1fr | 280px`, 3 rows: `50px topbar | 1fr chat | 84px inputbar`.
Left and right sidebars span rows 2–3. All panels animate in on load (`panel-in` keyframe).

### Topbar
- **Logo**: `AR` in `--a3` (light lavender), `IA` in `--teal` — no version number, no marketing pills
- **Connection dot**: single animated dot (green/amber/red) reflecting live model readiness
- **Right cluster**: Gen / KB / Skills / Memory counters in `--t3`/`--t2`
- **No pill badges** — pills were removed; they cluttered the topbar with redundant feature advertising

### Left Sidebar — Tab Bar
7 tabs (Config / Memory / Skills / Code / KB / Sessions / Cron) in a **2-row flex-wrap layout** (4 top, 3 bottom), each `flex: 1 1 25%`. This replaced a single-row layout that caused horizontal overflow at 256px width. Do not revert to single row without widening the sidebar.

### Left Sidebar — Config Tab
- Ollama URL input + model dropdown with live readiness check
- Connection status: `null` state (amber, "Checking model…") while `/api/ping` runs; green on ready, red on failure
- Quick Actions: Research / Evolution / New Session buttons
- Stats grid (2-col): Generation, Knowledge, Sessions, Skills, DB Size, Messages

### Main Chat Area
- Empty state: large `ARIA` logo (`AR` in `--a3`, `IA` in teal inline `<em>`), breathing animation, brief plain-text description — **no pills, no taglines**
- Messages: user bubbles right-aligned (max 74%), AI bubbles left (max 88%), avatar chips, footer with time/context tags
- Tool log: collapsible `<details>` block per message

### Input Bar
- Textarea (auto-grows, max 120px)
- Two **square icon buttons** (44×44px, zero padding, fixed width): `⟳` research, `⚡` evolve
- **Send →** button (44px tall, full `.btn` height) — all three buttons are the same height
- Previous design used `.btn-sm` for the icon buttons, making them shorter than Send; that was fixed

### Right Sidebar
- SOUL.md viewer (click to open full editor overlay)
- Evolution log (gen number, score, improvement summary)

## Intentional Design Decisions (preserve these)

- **No "v2" anywhere** — version numbers feel corny in a personal tool
- **No feature-advertising pills in the topbar or empty state** — the user knows what the tool does
- **No "100% local / zero cost" taglines** — accurate but belong in the README, not the UI
- **Center ARIA logo matches topbar exactly** — same `--a3` / teal split, same Bricolage Grotesque weight
- **Model readiness is real** — the dot reflects an actual `/api/ps` + generate-ping check, not just Ollama reachability
- **Tab bar wraps, not scrolls** — horizontal scroll in a fixed-width sidebar is a bad pattern
